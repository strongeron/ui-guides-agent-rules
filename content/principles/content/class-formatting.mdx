<Callout type="quote" title="Tailwind Golden Rule">
  Organize Tailwind classes in a consistent order: layout, positioning, sizing, spacing, typography, colors, effects, and variants last.
</Callout>

## Why This Matters

A consistent class order makes code scannable and predictable. When everyone follows the same pattern, it's easier to find specific utilities, spot duplicates, and understand component styling at a glance.

## Implementation

<CodeBlock title="Recommended Order" language="html">
{`<!-- Recommended class order -->
<div class="
  flex flex-col              /* layout */
  absolute top-0 left-0      /* positioning */
  w-full h-64                /* sizing */
  p-4 mt-2                   /* spacing */
  text-lg font-bold          /* typography */
  bg-primary text-white      /* colors */
  rounded-lg shadow-md       /* effects */
  hover:bg-primary-dark      /* variants */
">`}
</CodeBlock>
