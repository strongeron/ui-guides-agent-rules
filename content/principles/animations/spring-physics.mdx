<Callout type="quote" title="Natural Interactive Motion">
  Prefer spring-based animations for interactive elements (buttons, modals, drag). Configure mass/tension/friction instead of duration/easing for natural feel and interruptible motion.
</Callout>

## Why This Matters

Spring physics produce more natural-feeling motion than CSS easing curves. They respond to velocity changes, can be interrupted mid-animation, and settle naturally without hard stops.

Unlike duration-based animations, springs adapt to user interactions—dragging an element and releasing it will continue with the appropriate momentum.

<CodeBlock title="Spring Animation with motion/react">
```tsx
import { motion } from 'motion/react';

function SpringButton() {
  return (
    <motion.button
      whileTap={{ scale: 0.95 }}
      transition={{
        type: 'spring',
        stiffness: 400,
        damping: 17
      }}
    >
      Click me
    </motion.button>
  );
}
```
</CodeBlock>

<CodeBlock title="Spring Configuration Guide">
```tsx
// Quick, snappy response (buttons, toggles)
{ type: 'spring', stiffness: 400, damping: 17 }

// Gentle, smooth motion (modals, sheets)
{ type: 'spring', stiffness: 200, damping: 20 }

// Bouncy, playful motion (notifications)
{ type: 'spring', stiffness: 300, damping: 10 }
```
</CodeBlock>
