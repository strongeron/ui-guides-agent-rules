<Callout type="quote" title="Avoiding Animation Conflicts">
  When using Framer Motion/Motion with Tailwind, remove conflicting `transition-*` classes from animated elements. Let Motion handle transitions to prevent stuttery/weird motion.
</Callout>

## Why This Matters

Tailwind's `transition-*` classes apply CSS transitions. When combined with Framer Motion's JavaScript-based animations, they conflict and cause stuttering, double animations, or unexpected behavior.

Let Motion handle all animation properties through its `transition` prop instead.

## Implementation

<ExampleComparison
  badKey="animations-motion-tailwind-conflict-bad"
  goodKey="animations-motion-tailwind-conflict-good"
/>

<CodeBlock title="Wrong: Conflicting Classes">
```tsx
// Don't mix Tailwind transitions with Motion
<motion.div
  className="transition-all duration-300" // Conflicts!
  animate={{ opacity: 1, y: 0 }}
  initial={{ opacity: 0, y: 20 }}
>
  Content
</motion.div>
```
</CodeBlock>

<CodeBlock title="Right: Motion Handles Transitions">
```tsx
// Let Motion control all animation
<motion.div
  animate={{ opacity: 1, y: 0 }}
  initial={{ opacity: 0, y: 20 }}
  transition={{ duration: 0.3 }}
>
  Content
</motion.div>
```
</CodeBlock>

<CodeBlock title="Tailwind Classes That Conflict">
```css
/* Remove these from motion elements: */
transition-*
duration-*
ease-*
delay-*

/* These are safe to keep: */
hover:bg-*     /* Static state changes */
focus:ring-*   /* Focus styles */
```
</CodeBlock>
