<Callout type="quote" title="60fps Frame Budget">
  Animation work SHOULD complete within 16ms frame budget (60fps). Use requestAnimationFrame for JS animations. Batch DOM reads then writes to avoid layout thrashing in animation loops.
</Callout>

## Why This Matters

At 60fps, each frame has approximately 16.67ms for all work. Heavy JavaScript, layout recalculation, or paint work can cause dropped frames, resulting in janky animations.

Batch DOM reads before writes, use requestAnimationFrame, and keep animation logic minimal to stay within budget.

<CodeBlock title="Good Implementation">
```tsx
function SmoothAnimation() {
  const elementRef = useRef<HTMLDivElement>(null);

  useEffect(() => {
    let animationId: number;

    const animate = () => {
      // Batch reads first
      const rect = elementRef.current?.getBoundingClientRect();

      // Then batch writes
      if (elementRef.current && rect) {
        elementRef.current.style.transform = `translateX(${rect.left}px)`;
      }

      animationId = requestAnimationFrame(animate);
    };

    animationId = requestAnimationFrame(animate);
    return () => cancelAnimationFrame(animationId);
  }, []);

  return <div ref={elementRef}>Animated content</div>;
}
```
</CodeBlock>
