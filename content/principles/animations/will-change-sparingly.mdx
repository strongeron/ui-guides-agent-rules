<Callout type="quote" title="GPU Memory Best Practice">
  Use `will-change` sparingly—only on frequently animated elements. Apply via JS before animation starts, remove after completion. Never blanket-apply; each layer consumes GPU memory.
</Callout>

## Why This Matters

The `will-change` property creates a new compositor layer, consuming GPU memory. While it can improve animation performance by preparing the browser for changes, overusing it actually degrades performance.

Apply it just before an animation starts and remove it after completion. Never apply it to many elements at once or leave it on permanently.

## Implementation

<ExampleComparison
  badKey="animations-will-change-sparingly-bad"
  goodKey="animations-will-change-sparingly-good"
/>

<CodeBlock title="Good Implementation">
```tsx
function AnimatedCard() {
  const [isHovered, setIsHovered] = useState(false);

  return (
    <div
      className="card"
      style={{ willChange: isHovered ? 'transform' : 'auto' }}
      onMouseEnter={() => setIsHovered(true)}
      onMouseLeave={() => setIsHovered(false)}
    >
      {/* Content */}
    </div>
  );
}
```
</CodeBlock>

<CodeBlock title="CSS Alternative">
```css
/* Only apply will-change during interaction */
.card:hover {
  will-change: transform;
  transform: scale(1.05);
}

/* Never do this - wastes GPU memory */
.card {
  will-change: transform, opacity, filter; /* Bad! */
}
```
</CodeBlock>
