<Callout type="quote" title="Impact on LCP">
  Images are often the largest content element. Serving oversized images wastes bandwidth and delays LCP. A 2000px image on a 400px phone screen downloads 5x more data than needed.
</Callout>

## Why This Matters

Without responsive images, mobile users download desktop-sized images, wasting data and slowing load times. Modern browsers can select the optimal image from a set of sources based on:
- **Viewport width** - smaller images for mobile
- **Device pixel ratio** - 2x images for Retina displays
- **Image format support** - WebP/AVIF where supported

## Key Strategies

### 1. Use srcset with Width Descriptors
Tell the browser what image sizes are available.

<CodeBlock title="Width-based srcset" language="html">
{`<img
  src="/product-800.jpg"
  srcset="
    /product-400.jpg 400w,
    /product-800.jpg 800w,
    /product-1200.jpg 1200w,
    /product-1600.jpg 1600w
  "
  sizes="(max-width: 600px) 100vw, (max-width: 1200px) 50vw, 800px"
  alt="Product"
  width="800"
  height="600"
/>`}
</CodeBlock>

### 2. Use the picture Element for Art Direction
Different crops or images for different screen sizes.

<CodeBlock title="Art Direction with picture" language="html">
{`<picture>
  <!-- Mobile: square crop -->
  <source
    media="(max-width: 600px)"
    srcset="/hero-mobile.webp"
    type="image/webp"
  />
  <!-- Desktop: wide crop -->
  <source
    srcset="/hero-desktop.webp"
    type="image/webp"
  />
  <!-- Fallback -->
  <img src="/hero-desktop.jpg" alt="Hero" width="1200" height="600" />
</picture>`}
</CodeBlock>

### 3. Serve Modern Formats
WebP and AVIF offer 25-50% smaller files than JPEG/PNG with equal quality.

<CodeBlock title="Format Fallback Chain" language="html">
{`<picture>
  <!-- AVIF: best compression -->
  <source srcset="/image.avif" type="image/avif" />
  <!-- WebP: wide support -->
  <source srcset="/image.webp" type="image/webp" />
  <!-- JPEG: universal fallback -->
  <img src="/image.jpg" alt="..." width="800" height="600" />
</picture>`}
</CodeBlock>

### 4. Always Set Dimensions
Prevents CLS by reserving space. Use `aspect-ratio` in CSS as backup.

<CodeBlock title="Preventing CLS" language="html">
{`<!-- Explicit dimensions -->
<img src="/photo.jpg" width="800" height="600" alt="..." />

<!-- Or CSS aspect-ratio -->
<img src="/photo.jpg" class="w-full aspect-4/3 object-cover" alt="..." />`}
</CodeBlock>
