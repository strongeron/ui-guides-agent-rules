<Callout type="quote" title="Core Web Vital: LCP">
  Largest Contentful Paint (LCP) measures when the largest content element becomes visible. Target: under 2.5 seconds. Hero images are often the LCP element.
</Callout>

## Why This Matters

The Largest Contentful Paint is one of the three Core Web Vitals that Google uses for search ranking. Hero images, large text blocks, and videos are common LCP elements. Without proper optimization, users stare at a blank or partial page while the browser discovers and downloads these critical resources.

## Key Strategies

### 1. Preload LCP Images
Use `rel="preload"` for images that will be the LCP element. This tells the browser to fetch them immediately, before it parses CSS/JS.

<CodeBlock title="HTML Head" language="html">
{`<!-- Preload the hero image -->
<link
  rel="preload"
  href="/hero-image.webp"
  as="image"
  type="image/webp"
  fetchpriority="high"
/>

<!-- For responsive images, use imagesrcset -->
<link
  rel="preload"
  as="image"
  href="/hero-800.webp"
  imagesrcset="/hero-400.webp 400w, /hero-800.webp 800w, /hero-1200.webp 1200w"
  imagesizes="(max-width: 600px) 400px, 800px"
/>`}
</CodeBlock>

### 2. Never Lazy-Load Above the Fold
Lazy loading (`loading="lazy"`) delays image loading until near the viewport. For LCP elements, this adds unnecessary delay.

<CodeBlock title="Image Loading Strategy" language="html">
{`<!-- Bad: lazy-loading the hero image -->
<img src="/hero.webp" loading="lazy" alt="Hero" />

<!-- Good: eager loading with high priority -->
<img
  src="/hero.webp"
  loading="eager"
  fetchpriority="high"
  decoding="async"
  alt="Hero"
/>`}
</CodeBlock>

### 3. Use fetchpriority="high"
The `fetchpriority` attribute explicitly tells the browser which resources matter most.
