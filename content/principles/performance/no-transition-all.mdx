<Callout type="quote" title="Tailwind Golden Rule">
  Never use transition-all. Explicitly specify which properties to transition for better performance and predictable behavior.
</Callout>

## Why This Matters

`transition-all` animates every CSS property that changes, including layout-triggering properties you didn't intend to animate. This causes:

- Unexpected animations (background-color, border transitions you didn't want)
- Performance issues (animating layout properties)
- Harder debugging (unclear what's actually transitioning)

## Implementation

<CodeBlock title="Tailwind Classes" language="html">
{`<!-- Bad: Transitions everything -->
<button class="transition-all hover:bg-primary hover:scale-105">

<!-- Good: Explicit transition properties -->
<button class="transition-transform hover:scale-105">

<!-- Good: Multiple explicit properties -->
<button class="transition-[transform,box-shadow] hover:scale-105 hover:shadow-lg">`}
</CodeBlock>
