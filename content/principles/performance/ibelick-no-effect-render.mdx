<Callout type="quote" title="UI Skills Rule">
  NEVER use useEffect for render logic. If a value can be computed from props or state, compute it during render instead.
</Callout>

## Why This Matters

Using useEffect to set state based on other state causes:
- Extra re-renders (render → effect → setState → render)
- Potential for infinite loops
- Harder to reason about data flow
- Worse performance

Instead, compute derived values directly or use useMemo for expensive calculations.
