<Callout type="quote" title="Tailwind Golden Rule">
  Always configure content paths to include all files that use Tailwind classes. Missing paths mean unused CSS in production or missing styles.
</Callout>

## Why This Matters

Tailwind CSS v4 uses `@source` directive (or v3's `content` config) to scan files for class names. If a file isn't included in these paths, its classes won't be included in the production build. Conversely, overly broad globs can slow down builds or include unintended files.

## Implementation

<CodeBlock title="tailwind.config.js (v3)" language="javascript">
{`// Bad: Too narrow - misses component files
module.exports = {
  content: ['./src/pages/**/*.{js,jsx}'],
}

// Good: Comprehensive coverage
module.exports = {
  content: [
    './src/**/*.{js,ts,jsx,tsx,mdx}',
    './components/**/*.{js,ts,jsx,tsx}',
  ],
}`}
</CodeBlock>

<CodeBlock title="CSS with @source (v4)" language="css">
{`/* Tailwind v4 uses @source directive */
@import "tailwindcss";
@source "../src/**/*.{js,ts,jsx,tsx}";
@source "../components/**/*.tsx";`}
</CodeBlock>
