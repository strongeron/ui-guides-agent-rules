<Callout type="quote" title="Tailwind Golden Rule">
  Keep content paths specific and safelist minimal. Overly broad configurations slow builds and bloat CSS output.
</Callout>

## Why This Matters

Tailwind's purge/content scanning determines what CSS ships to production. Poor configuration leads to:

- **Overly broad globs**: Scanning node_modules or unrelated files slows builds
- **Missing paths**: Required classes purged from production
- **Overused safelist**: Defeats the purpose of tree-shaking

## Implementation

<CodeBlock title="tailwind.config.js" language="javascript">
{`// Bad: Too broad, slow builds
module.exports = {
  content: ['./**/*.{js,ts,jsx,tsx}'], // Includes node_modules!
  safelist: ['bg-red-500', 'bg-blue-500', ...], // Safelisting everything
}

// Good: Specific paths, minimal safelist
module.exports = {
  content: [
    './src/**/*.{js,ts,jsx,tsx}',
    './components/**/*.tsx',
  ],
  safelist: [
    // Only safelist truly dynamic classes (e.g., from CMS)
    { pattern: /^prose-/ },
  ],
}`}
</CodeBlock>
