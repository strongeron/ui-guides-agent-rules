<Callout type="quote" title="Tailwind Golden Rule">
  Avoid arbitrary values like p-[17px] or text-[#1a1a1a]. Use theme tokens for consistency and smaller CSS output.
</Callout>

## Why This Matters

Arbitrary values (`[17px]`, `[#ff5500]`) create one-off utility classes that:

- Bypass your design system's spacing/color scale
- Create inconsistencies across the codebase
- Generate additional CSS (not reusable)
- Make maintenance harder (magic numbers scattered)

## Implementation

<CodeBlock title="Tailwind Classes" language="html">
{`<!-- Bad: Arbitrary values -->
<div class="p-[17px] text-[#1a1a1a] w-[327px]">

<!-- Good: Theme tokens -->
<div class="p-4 text-foreground w-80">

<!-- If you need custom values, extend the theme -->
// tailwind.config.js
theme: {
  extend: {
    spacing: { '18': '4.5rem' }
  }
}`}
</CodeBlock>
