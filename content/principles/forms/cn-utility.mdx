<Callout type="quote" title="Tailwind Golden Rule">
  Create a cn() utility that combines clsx (conditional classes) with tailwind-merge (conflict resolution). This is the industry-standard pattern for React + Tailwind.
</Callout>

## Why This Matters

clsx makes conditional classes easy but doesn't resolve conflicts. tailwind-merge resolves conflicts but isn't as ergonomic for conditionals. Combining them gives you the best of both worlds - a single utility for all class needs.

## Implementation

<CodeBlock title="lib/utils.ts" language="typescript">
{`import { type ClassValue, clsx } from 'clsx';
import { twMerge } from 'tailwind-merge';

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}

// Usage in components
<button
  className={cn(
    'px-4 py-2 rounded-lg',
    isActive && 'bg-primary text-primary-foreground',
    isDisabled && 'opacity-50 cursor-not-allowed',
    className
  )}
/>`}
</CodeBlock>
