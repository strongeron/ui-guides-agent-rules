<Callout type="quote" title="Tailwind Golden Rule">
  Use the sr-only utility for content that should be read by screen readers but hidden visually. Never use display:none or visibility:hidden for accessible content.
</Callout>

## Why This Matters

Screen reader users need context that sighted users get visually. Icon-only buttons need labels, status indicators need descriptions. The `sr-only` class hides content visually while keeping it in the accessibility tree.

## Implementation

<CodeBlock title="Tailwind Classes" language="html">
{`<!-- Icon button with sr-only label -->
<button aria-label="Close">
  <XIcon aria-hidden="true" />
  <span class="sr-only">Close dialog</span>
</button>

<!-- Status with visual and screen reader text -->
<div class="flex items-center gap-2">
  <span class="w-2 h-2 bg-green-500 rounded-full" aria-hidden="true" />
  <span class="sr-only">Status:</span>
  Online
</div>

<!-- Table with sr-only headers -->
<th class="sr-only">Actions</th>`}
</CodeBlock>
