<Callout type="quote" title="WCAG 2.5.5 - Level AAA">
  The size of the target for pointer inputs is at least 44 by 44 CSS pixels. Small touch targets cause errors for users with motor impairments, tremors, or when using devices in motion.
</Callout>

## Why This Matters

Small touch targets are hard to tap accurately, especially for users with motor impairments, users in motion, or anyone using a phone one-handed. WCAG requires 44x44px minimum, but 48x48px (Apple/Google recommendation) is better. The visual element can be smaller - padding can extend the touch area.

## Implementation

<CodeBlock title="CSS/Tailwind" language="html">
{`<!-- Icon button with adequate touch target -->
<button className="p-3 min-w-[44px] min-h-[44px]">
  <Icon className="w-5 h-5" />
</button>

<!-- Small icon, large hit area via padding -->
<button className="p-2.5">
  <Icon className="w-6 h-6" />
  <!-- Total: 24px + 20px padding = 44px -->
</button>

<!-- Link with adequate target -->
<a href="/page" className="inline-flex items-center min-h-[44px] px-4">
  View details
</a>

<!-- Checkbox with label extending hit area -->
<label className="flex items-center gap-2 p-2 min-h-[44px] cursor-pointer">
  <input type="checkbox" />
  <span>Accept terms</span>
</label>`}
</CodeBlock>
