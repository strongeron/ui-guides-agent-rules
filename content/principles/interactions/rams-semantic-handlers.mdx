<Callout type="quote" title="WCAG 4.1.2 - Level A">
  User interface components must have accessible names and roles. Adding click handlers to divs and spans creates custom widgets that lack the built-in accessibility of native elements.
</Callout>

## Why This Matters

Native interactive elements (button, a, input) come with built-in accessibility: keyboard support, focus management, and proper ARIA roles. When you put onClick on a div, you get none of this. You'd need to add role, tabIndex, keyboard handlers, and focus styling manually - and often something gets missed.

## Implementation

<CodeBlock title="HTML/JSX" language="html">
{`<!-- Use button for actions -->
<button onClick={handleDelete}>Delete</button>

<!-- Use anchor for navigation -->
<a href="/settings">Settings</a>

<!-- If you MUST use a div (rarely needed), add everything -->
<div
  role="button"
  tabIndex={0}
  onClick={handleAction}
  onKeyDown={(e) => {
    if (e.key === 'Enter' || e.key === ' ') {
      e.preventDefault();
      handleAction();
    }
  }}
  aria-label="Perform action"
>
  Custom interactive
</div>

<!-- But really, just use a button -->
<button className="custom-styles">
  Much simpler
</button>`}
</CodeBlock>
