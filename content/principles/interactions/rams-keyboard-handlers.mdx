<Callout type="quote" title="WCAG 2.1.1 - Level A">
  All functionality must be operable through a keyboard. Elements with onClick handlers must also respond to keyboard events (Enter/Space) or use semantic elements like button that handle this automatically.
</Callout>

## Why This Matters

Many users cannot use a mouse - they navigate with keyboards, switch devices, or voice control. If an interactive element only responds to clicks, these users are locked out. Native buttons handle keyboard events automatically; custom click handlers on divs/spans do not.

## Implementation

<CodeBlock title="HTML/JSX" language="html">
{`<!-- Best: Use semantic button - keyboard handled automatically -->
<button onClick={handleClick}>
  Submit
</button>

<!-- If you must use a div, add keyboard support -->
<div
  role="button"
  tabIndex={0}
  onClick={handleClick}
  onKeyDown={(e) => {
    if (e.key === 'Enter' || e.key === ' ') {
      e.preventDefault();
      handleClick();
    }
  }}
>
  Custom button
</div>

<!-- For links, use anchor tags -->
<a href="/page" onClick={handleNavigation}>
  Navigate
</a>`}
</CodeBlock>
