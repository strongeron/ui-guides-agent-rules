<Callout type="quote" title="WCAG 2.4.7 - Level AA">
  Any keyboard operable user interface must have a mode where the keyboard focus indicator is visible. Removing or hiding focus outlines makes keyboard navigation impossible.
</Callout>

## Why This Matters

Keyboard users need to know which element is currently focused. Without visible focus indicators, they're navigating blind - like using a mouse with an invisible cursor. Never remove focus outlines without providing an alternative visible indicator.

## Implementation

<CodeBlock title="CSS/Tailwind" language="css">
{`/* Never do this without replacement */
/* :focus { outline: none; } */

/* Use focus-visible for keyboard-only focus */
.button:focus-visible {
  outline: 2px solid var(--ring);
  outline-offset: 2px;
}

/* Tailwind approach */
<button className="focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2">
  Click me
</button>

/* Custom focus style is fine, just make it visible */
.custom-focus:focus-visible {
  box-shadow: 0 0 0 3px rgba(59, 130, 246, 0.5);
}`}
</CodeBlock>
