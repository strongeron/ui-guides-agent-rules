<Callout type="quote" title="UI Skills Rule">
  Avoid rebuilding keyboard navigation and focus behavior. Accessible primitive libraries handle edge cases you'll miss.
</Callout>

## Why This Matters

Manual keyboard handling is error-prone. You'll likely miss:
- RTL text direction handling
- Disabled item navigation
- Type-ahead search in lists
- Focus restoration after close
- Screen reader live regions
- Roving tabindex patterns

Let battle-tested libraries handle this complexity.
