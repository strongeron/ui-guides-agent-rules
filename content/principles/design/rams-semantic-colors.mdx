<Callout type="quote" title="Design System Principle">
  Semantic colors (success, warning, error, info) must have consistent meaning. Green always means success. Red always means error or danger. Breaking this convention confuses users.
</Callout>

## Why This Matters

Users learn color meanings quickly: green = good/success, red = danger/error, yellow/orange = warning, blue = info. Using red for a success state or green for a delete button violates learned conventions and causes confusion or errors.

## Implementation

<CodeBlock title="CSS/Tailwind" language="html">
{`<!-- Consistent semantic colors -->
<Badge className="bg-success text-success-foreground">Active</Badge>
<Badge className="bg-destructive text-destructive-foreground">Inactive</Badge>
<Badge className="bg-warning text-warning-foreground">Pending</Badge>

<!-- Destructive actions are red -->
<Button variant="destructive">Delete</Button>

<!-- Success feedback is green -->
<Toast className="bg-success">Saved successfully!</Toast>

<!-- NEVER: Green delete button -->
<!-- NEVER: Red success message -->
<!-- NEVER: Same color for opposite meanings -->`}
</CodeBlock>
