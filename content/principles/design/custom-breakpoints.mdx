<Callout type="quote" title="Tailwind Golden Rule">
  Use Tailwind's default breakpoints (sm, md, lg, xl, 2xl) unless your design requires specific values. Add custom breakpoints in theme.extend when needed.
</Callout>

## Why This Matters

Default breakpoints work for most projects, but some designs need specific widths (e.g., matching a design system's grid). Custom breakpoints should be added via `theme.extend.screens`, not arbitrary values like `min-[960px]:`.

## Implementation

<CodeBlock title="tailwind.config.js" language="javascript">
{`// Bad: Arbitrary breakpoints everywhere
<div class="min-[960px]:flex min-[1200px]:grid">

// Good: Named custom breakpoints
module.exports = {
  theme: {
    extend: {
      screens: {
        'tablet': '960px',
        'desktop': '1200px',
      }
    }
  }
}
// Then use: tablet:flex desktop:grid`}
</CodeBlock>
