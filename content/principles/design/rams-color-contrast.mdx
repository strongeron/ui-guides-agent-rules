<Callout type="quote" title="WCAG 1.4.11 - Level AA">
  Non-text UI components (buttons, form fields, icons) need at least 3:1 contrast ratio against adjacent colors. This ensures interactive elements are visible and distinguishable.
</Callout>

## Why This Matters

Low contrast UI elements are hard to see, especially for users with low vision, in bright environments, or on low-quality screens. This includes form field borders, button backgrounds, icons, and focus indicators. If users can't see an element, they can't interact with it.

## Implementation

<CodeBlock title="CSS/Tailwind" language="html">
{`<!-- Button with good contrast -->
<button className="bg-primary text-primary-foreground">
  Submit
</button>

<!-- Input with visible border -->
<input className="border-2 border-border" />
<!-- NOT: border-gray-200 (too light) -->

<!-- Icon with adequate contrast -->
<Icon className="text-foreground" />
<!-- NOT: text-gray-300 -->

<!-- Focus indicator with strong contrast -->
<button className="focus-visible:ring-2 focus-visible:ring-ring">
  Action
</button>`}
</CodeBlock>
