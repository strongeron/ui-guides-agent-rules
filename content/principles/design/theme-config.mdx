<Callout type="quote" title="Tailwind Golden Rule">
  Always use theme.extend to add custom values. Overwriting top-level theme keys removes all default values.
</Callout>

## Why This Matters

When you define `theme.colors` directly, you replace all of Tailwind's default colors. Using `theme.extend.colors` adds your custom colors while keeping the defaults like `slate`, `gray`, `red`, etc.

## Implementation

<CodeBlock title="tailwind.config.js" language="javascript">
{`// Bad: Overwrites ALL default colors
module.exports = {
  theme: {
    colors: {
      brand: '#0066cc',
    }
  }
}
// Result: Only 'brand' exists, no 'red', 'blue', etc.

// Good: Extends defaults
module.exports = {
  theme: {
    extend: {
      colors: {
        brand: '#0066cc',
      }
    }
  }
}
// Result: 'brand' + all default colors`}
</CodeBlock>
