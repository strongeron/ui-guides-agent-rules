## Decision Criteria

A standalone rule should become a principle if:
1. **Demonstrable**: Can be shown visually with Good/Bad examples
2. **Actionable**: Developers can clearly follow the guidance
3. **Sourced**: Has backing documentation from Vercel or web standards
4. **Impactful**: Significantly affects user experience or accessibility

A rule should remain standalone if:
1. Too abstract to demonstrate visually
2. Only relevant in specific contexts
3. Serves as supplementary guidance to existing principles
